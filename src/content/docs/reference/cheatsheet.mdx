## 核心命令

| 命令 | 功能 |
|------|------|
| `/help` | 上下文帮助 |
| `/clear` | 重置对话 |

| `/status` | 会话状态 + 上下文用量 |

| `/batch` | 通过并行 worktree agent 进行大规模重构 |
| `/teleport` | 从 Web 传送会话 |
| `/tasks` | 监控后台任务 |
| `/remote-control` | 启动远程控制会话（Pro/Max） |
| `/fast` | 切换快速模式（2.5 倍速度，6 倍成本） |
| `/debug` | 系统化故障排查 |
| `/exit` | 退出（或 Ctrl+D） |

## 上下文管理

<Aside type="caution">
上下文管理至关重要。关注状态栏中的 `Ctx(u):`。超过 70% = `/compact`，超过 85% = `/clear`。
</Aside>

### 上下文阈值

| 上下文 % | 状态 | 操作 |
|----------|------|------|
| 0-50% | 绿色 | 自由工作 |
| 50-70% | 黄色 | 谨慎选择 |
| 70-90% | 橙色 | 立即 `/compact` |
| 90%+ | 红色 | 必须 `/clear` |

### 按症状操作

| 现象 | 操作 |
|------|------|
| 回复变短 | `/compact` |
| 频繁遗忘 | `/clear` |
| 上下文 >70% | `/compact` |
| 任务完成 | `/clear` |

### 上下文恢复命令

| 命令 | 用途 |
|------|------|
| `/compact` | 总结并释放上下文 |
| `/clear` | 重新开始 |
| `/rewind` | 撤销最近的变更 |
| `claude -c` | 恢复上次会话 |
| `claude -r <id>` | 恢复指定会话 |

### 状态栏

```
Model: Sonnet | Ctx: 89.5k | Cost: $2.11 | Ctx(u): 56.0%
```

**增强状态栏（[ccstatusline](https://github.com/sirmalloc/ccstatusline)）：** 添加到 `~/.claude/settings.json`：
```json
{ "statusLine": { "type": "command", "command": "npx -y ccstatusline@latest", "padding": 0 } }
```

## 内部机制速览

| 概念 | 要点 |
|------|------|
| **主循环** | 简单的 `while(tool_call)` —— 无 DAG、无分类器 |
| **工具** | 8 个核心工具：Bash、Read、Edit、Write、Grep、Glob、Task、TodoWrite |
| **上下文** | ~200K tokens，75-92% 时自动压缩 |
| **子 Agent** | 隔离上下文，最大深度=1 |
| **理念** | 「少搭架子，多信模型」—— 信任 Claude 的推理能力 |

## 典型工作流

<Steps>
1. 启动会话：`claude`
2. 检查上下文：`/status`
3. 复杂任务进入计划模式：`Shift+Tab` x 2
4. 描述任务：清晰、具体的提示词（做什么、在哪里、怎么做、如何验证）
5. 审查变更：务必阅读 diff
6. 接受/拒绝：`y/n`
7. 验证：运行测试
8. 任务完成后提交
9. 上下文 >70% 时执行 `/compact`
</Steps>

## 计划模式与思考

| 功能 | 激活方式 | 用途 |
|------|---------|------|
| **计划模式** | `Shift+Tab` x 2 或 `/plan` | 只探索不修改 |
| **OpusPlan** | `/model opusplan` | Opus 负责规划，Sonnet 负责执行 |

<Aside type="tip">
Opus 4.6 默认以最大预算**开启思考模式**。"think hard"或"ultrathink"等关键词仅是装饰，不会实际控制深度。
</Aside>

| 控制方式 | 操作 | 持久性 |
|---------|------|--------|
| **Alt+T** | 开关思考模式 | 当前会话 |
| **/config** | 全局启用/禁用 | 永久 |
| **`/model` 滑块** | 左右切换：`low\|medium\|high` | 当前会话 |
| **`CLAUDE_CODE_EFFORT_LEVEL`** | 环境变量：`low\|medium\|high` | Shell 会话 |

### 快速模型选择

| 任务 | 模型 | 力度 |
|------|------|------|
| 重命名、模板代码、测试生成 | Haiku | low |
| 功能开发、调试、重构 | Sonnet | medium-high |
| 架构设计、安全审计 | Opus | high-max |

### 动态模型切换

**模式**：Sonnet 起步（速度） -> 切换 Opus（复杂度） -> 回到 Sonnet

```bash
# 会话开始（默认 Sonnet）
claude

# 遇到复杂功能
/model opus                    # 切换到深度推理

# 功能完成，回归日常
/model sonnet                  # 速度 + 成本优化
```

**最佳实践**：
- 在**任务边界**切换，而非任务中间
- Opus 用于：架构决策、复杂调试、安全关键代码
- Sonnet 用于：日常编辑、重构、测试编写
- Haiku 用于：简单修复、拼写错误、验证检查

### 成本对照

| 模型 | 输入 | 输出 | 使用场景 |
|------|------|------|---------|
| Opus | $15/MTok | $75/MTok | 复杂推理（10-20% 的任务） |
| Sonnet | $3/MTok | $15/MTok | 大部分开发（70-80% 的任务） |
| Haiku | $0.25/MTok | $1.25/MTok | 简单验证（5-10% 的任务） |

## 远程控制 — 移动端访问（v2.1.51+）

<Aside type="caution">
仅 Pro/Max 用户可用 —— Team、Enterprise 或 API Key 不支持。
</Aside>

```bash
# 从终端启动（新会话）
claude remote-control

# 或在活动会话中：
/rc        # （或 /remote-control）
```

**从手机/平板/浏览器连接：**

<Steps>
1. 扫描 **二维码**（启动后按空格键显示）
2. 或在浏览器 / Claude 移动应用中打开 **会话 URL**
3. 或使用 `/mobile` 获取 App Store / Play Store 下载链接
</Steps>

| 已知限制 | 说明 |
|----------|------|
| 同时仅 1 个会话 | 仅支持一个远程会话 |
| 斜杠命令失效 | `/new`、`/compact` 在远程端变为纯文本，需在本地终端执行 |
| 终端必须保持开启 | 关闭本地终端会结束会话 |
| 网络超时 | ~10 分钟断连后会话过期 |

**高级用法：tmux 多会话**（绕过单会话限制）：
```bash
tmux new-session -s dev
# 每个窗格 = 独立的 claude 会话
# 在需要远程控制的窗格中运行 /rc
```

## 任务管理（v2.1.16+）

**两套系统可用：**

| 系统 | 适用场景 | 持久化 |
|------|---------|--------|
| **Tasks API**（v2.1.16+） | 多会话项目、依赖关系 | 磁盘（`~/.claude/tasks/`） |
| **TodoWrite**（旧版） | 简单单会话 | 仅会话内 |

### Tasks API 命令

```bash
# 启用跨会话持久化
export CLAUDE_CODE_TASK_LIST_ID="project-name"
claude

# 在 Claude 中：创建任务层次
> "Create tasks for auth system with dependencies"

# 稍后恢复（新会话）
export CLAUDE_CODE_TASK_LIST_ID="project-name"
claude
> "TaskList to see current state"
```

<Aside type="note">
TaskList 仅显示 `id`、`subject`、`status`、`blockedBy`。查看 `description`/`metadata` 需使用 `TaskGet(taskId)`。
</Aside>

```
简单任务      → 直接问 Claude
复杂任务      → 先用 Tasks API 规划
高风险变更    → 先用计划模式
```

## 常见问题速修

| 问题 | 解决方案 |
|------|---------|
| "Command not found" | 检查 PATH，重装 npm 全局包 |
| 上下文过高（>70%） | 立即 `/compact` |
| 响应变慢 | `/compact` 或 `/clear` |
| MCP 不工作 | `claude mcp list`，检查配置 |
| 权限被拒绝 | 检查 `settings.local.json` |
| Hook 阻塞 | 检查 Hook 退出码，审查逻辑 |

**健康检查：**
```bash
which claude && claude doctor && claude mcp list
```

## 黄金法则

<Steps>
1. **务必审查 diff** 再接受变更
2. **及时 `/compact`** 上下文超过 70% 之前
3. **描述要具体**（做什么、在哪里、怎么做、如何验证）
4. **复杂/高风险任务先用计划模式**
5. **为每个项目创建 CLAUDE.md**
6. **每完成一个任务就提交**
7. **了解发送了什么** —— 提示词、文件、MCP 结果都会发送到 Anthropic（[隐私设置](https://claude.ai/settings/data-privacy-controls)）
</Steps>

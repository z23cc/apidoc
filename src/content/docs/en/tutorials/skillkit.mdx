## What is SkillKit?

SkillKit is the **open source package manager** for AI agent skills. Different agents use different formats — Claude Code uses `.claude/skills/`, Cursor uses `.mdc`, Copilot uses `.github/skills/`. SkillKit lets you **write a skill once, deploy it to 44 agents**.

## Quick Start

<Steps>
1. Initialize your project:
   ```bash
   cd my-project
   skillkit init
   ```
   `init` auto-detects your AI agent, creates the skills folder, and saves preferences.

2. Get recommendations:
   ```bash
   skillkit recommend
   ```

3. Install skills:
   ```bash
   skillkit install anthropics/skills
   ```

4. Deploy and sync:
   ```bash
   skillkit sync
   ```
</Steps>

### What init creates

| Agent | Skills Directory | Config |
|-------|-----------------|--------|
| Claude Code | `.claude/skills/` | `.claude/config.json` |
| Cursor | `.cursor/skills/` | `.cursor/config.mdc` |
| Codex | `.codex/skills/` | `AGENTS.md` |
| GitHub Copilot | `.github/skills/` | — |
| Universal | `skills/` | `AGENTS.md` |

```bash
skillkit init --agent cursor   # Specify agent
skillkit init --list           # Show all 44 supported agents
skillkit init --quiet          # Silent mode (for CI)
```

## Skill Lifecycle

Skills progress through 6 stages:

| Stage | Description |
|-------|-------------|
| Discovery | Scans directories for SKILL.md files |
| Quality Evaluation | Produces 0-100 scores (grades A-F) |
| Parsing | Extracts YAML frontmatter and Markdown content |
| Translation | Converts to agent-specific formats |
| Installation | Deploys to agent directories with metadata tracking |
| Testing | 14+ assertion types validate skills |

### Quality Evaluation

Skills are scored across four weighted dimensions:

| Dimension | Points | What It Measures |
|-----------|--------|-----------------|
| Structure | 25 | Frontmatter completeness, markdown sections |
| Clarity | 30 | Description quality, instruction precision |
| Specificity | 30 | Concrete examples, actionable steps |
| Advanced | 15 | Test coverage, version info, examples |

Grades: A (90-100), B (80-89), C (70-79), D (60-69), F (below 60)

## Marketplace

```bash
skillkit marketplace --tags typescript,testing  # Filter by tags

```

### Install from Marketplace

```bash
skillkit install anthropics/skills                        # Full repo
skillkit install anthropics/skills --skills pdf,xlsx      # Specific skills
skillkit install anthropics/skills --agent claude-code,cursor  # Specific agents
```

### Install Sources

```bash
skillkit install anthropics/skills          # GitHub
skillkit install gitlab:team/skills         # GitLab
skillkit install ./my-local-skills          # Local path
```

### Publish Skills

**Self-host (Recommended)** — generate RFC 8615 well-known URI structure:

```bash
skillkit create my-skill
skillkit validate my-skill
skillkit publish ./my-skill --output ./public
```

Creates:

```
.well-known/skills/
  index.json              # Skill manifest for auto-discovery
  my-skill/
    SKILL.md
```

Deploy to your domain. Users install via `skillkit add https://your-domain.com`.

**Submit to marketplace**:

```bash
skillkit publish submit    # Opens GitHub issue for review
```

## Cross-platform Translation

SkillKit uses an adapter pattern to auto-convert between 44 agent formats:

```bash
skillkit translate react-patterns --to cursor     # Single skill
skillkit translate --all --to windsurf            # All skills
skillkit translate my-skill --to codex --dry-run  # Preview mode
```

### Batch Translation

```bash
skillkit agent translate --source ./my-skills --to cursor
skillkit agent translate --source ./skills --to cursor --recursive
```

### Format Mapping

| Source | Target | Notes |
|--------|--------|-------|
| SKILL.md | .mdc | Cursor format |
| SKILL.md | Markdown | GitHub Copilot |
| .mdc | SKILL.md | Reverse conversion |

## Workflow Orchestration

Compose skills into multi-step processes:

```bash
skillkit workflow list           # List workflows
skillkit workflow run <name>     # Run workflow
skillkit workflow create <name>  # Create workflow
```

### Workflow Definition

```yaml
name: feature-development
description: Complete feature development workflow
steps:
  - name: plan
    skill: planning
  - name: implement
    skill: coding-standards
  - name: test
    skill: testing-patterns
  - name: review
    skill: code-review
```

### Conditional and Parallel Execution

```yaml
steps:
  # Conditional step
  - name: security
    skill: security-audit
    condition: "files.any('**/*.ts')"

  # Parallel steps
  - name: checks
    parallel:
      - skill: linting
      - skill: testing
```

## Testing Framework

Built-in testing with 14+ assertion types:

```bash
skillkit test              # Run all tests
skillkit test ./my-skill   # Test specific skill
skillkit test --tags unit  # Filter by tag
skillkit test --json       # CI output format
```

### Test File

Create `test.yaml` alongside your skill:

```yaml
name: my-skill-tests
skill: my-skill
tests:
  - name: validates input
    input: |
      Process this file
    assertions:
      - contains: "processed"
      - not_contains: "error"
```

### Assertion Types

| Assertion | Description |
|-----------|-------------|
| `contains` | Text present |
| `not_contains` | Text absent |
| `matches` | Regex match |
| `starts_with` / `ends_with` | Prefix/suffix check |
| `has_section` | Markdown section exists |
| `has_code_block` | Code block with language |
| `word_count_min` / `max` | Word count range |
| `no_todos` | No TODO comments |
| `has_frontmatter` | YAML frontmatter present |

## REST API

Start an HTTP server for runtime skill discovery:

```bash
skillkit serve                    # Start at http://localhost:3737
skillkit serve --port 8080        # Custom port
```

### Key Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/search?q=...` | Search skills |
| POST | `/search` | Search with filters |
| GET | `/skills/:owner/:repo/:id` | Get specific skill |
| GET | `/trending?limit=20` | Trending skills |
| GET | `/categories` | Skill categories |
| GET | `/health` | Server health check |
| GET | `/docs` | Swagger UI |
| GET | `/openapi.json` | OpenAPI spec |

### Ranking Algorithm

| Signal | Points | What It Measures |
|--------|--------|-----------------|
| Content | 40 | Has description and SKILL.md body |
| Query match | 30 | Name/description match to search |
| Popularity | 15 | GitHub stars + installs (log-scaled) |
| References | 15 | Has example files, docs, or resources |

Rate limit: 60 requests/minute/IP. Built-in LRU memory cache.

## Configuration

SkillKit uses 5-layer configuration (lowest to highest priority):

1. Default values in code
2. Global user preferences (`~/.skillkit/preferences.json`)
3. Project settings (`skillkit.yaml`)
4. Environment variables
5. CLI flags

### Project Configuration

Create `skillkit.yaml` in your project root:

```yaml
version: 1
agent: cursor              # Override auto-detection
autoSync: true             # Sync on changes
enabledSkills:
  - pdf
  - xlsx
  - react-patterns
sources:
  - anthropics/skills
  - vercel-labs/agent-skills
```

### Environment Variables

| Variable | Purpose |
|----------|---------|
| `SKILLKIT_AGENT` | Override default agent |
| `SKILLKIT_DIR` | Custom SkillKit directory |
| `SKILLKIT_DEBUG` | Enable debug logging |

### Configuration Profiles

```yaml
profiles:
  development:
    agent: claude
    autoSync: true
  production:
    agent: universal
    autoSync: false
```

```bash
skillkit --profile production sync
```

## Chrome Extension

Save any webpage as a skill from your browser:

<Steps>
1. Build: `pnpm --filter @skillkit/extension build`
2. Chrome → `chrome://extensions` → Load unpacked → select `packages/extension/dist/`
3. Click the extension icon or right-click → "Save page as Skill"
</Steps>

The extension sends the page URL to the SkillKit API for server-side extraction. The resulting SKILL.md downloads automatically. Then run `skillkit install ~/Downloads/skillkit-skills/my-skill` to deploy.

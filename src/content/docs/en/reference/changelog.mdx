## 2.1.63

**Highlights**: Built-in slash commands, HTTP Hooks, extensive memory leak fixes

- Added `/simplify` and `/batch` bundled slash commands
- Project configs & auto memory now shared across git worktrees of the same repository
- Added `ENABLE_CLAUDEAI_MCP_SERVERS=false` env var to opt out from making claude.ai MCP servers available
- Improved `/model` command to show the currently active model in the slash command menu
- Added HTTP hooks, which can POST JSON to a URL and receive JSON instead of running a shell command
- Added manual URL paste fallback during MCP OAuth authentication
- Added "Always copy full response" option to the `/copy` picker
- VSCode: Added session rename and remove actions to the sessions list
- Fixed multiple memory leaks (bridge polling, MCP OAuth, hooks config, bash prefix cache, git root detection cache, JSON parsing cache, etc.)
- Fixed `/clear` not resetting cached skills
- Fixed a race condition in the REPL bridge message ordering

## 2.1.51

**Highlights**: Remote Control, plugin npm registries, performance

- Added `claude remote-control` subcommand for external builds
- Updated plugin marketplace default git timeout from 30s to 120s, added `CLAUDE_CODE_PLUGIN_GIT_TIMEOUT_MS`
- Added support for custom npm registries and specific version pinning for plugins
- BashTool now skips login shell (`-l` flag) by default, improving performance
- Tool results larger than 50K characters now persisted to disk (previously 100K)
- `/model` picker now shows human-readable labels (e.g., "Sonnet 4.5")
- Managed settings can now be set via macOS plist or Windows Registry
- Fixed a security issue where `statusLine` and `fileSuggestion` hook commands could execute without workspace trust

## 2.1.50

- Added `WorktreeCreate` and `WorktreeRemove` hook events

## 2.1.49

**Highlights**: Worktree isolation, Ctrl+F agent kill, plugin settings

- Added `--worktree` (`-w`) flag to start Claude in an isolated git worktree
- Subagents support `isolation: "worktree"` for working in temporary git worktrees
- Added Ctrl+F keybinding to kill background agents (two-press confirmation)
- Agent definitions support `background: true` to always run as background tasks
- Plugins can ship `settings.json` for default configuration
- Simple mode now includes the file edit tool alongside Bash
- Sonnet 4.5 with 1M context removed from Max plan in favor of Sonnet 4.6
- Added `ConfigChange` hook event for configuration change auditing
- SDK model info now includes `supportsEffort`, `supportedEffortLevels`, and `supportsAdaptiveThinking` fields
- Fixed multiple unbounded memory growth issues (tree-sitter, Yoga WASM)

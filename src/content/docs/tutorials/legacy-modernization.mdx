## 为什么遗留代码现代化如此困难？

遗留代码迁移的真正成本不在于迁移本身 -- 而在于**发现阶段**。原始开发者已经退休，文档缺失或过时，代码被不了解全局的工程师修补了数十年。搞清楚什么与什么通信，需要按小时计费的咨询顾问。

AI 改变了经济模型，因为它恰好能自动化这个阶段。

<Aside type="tip" title="规模参考">
- 约 2200 亿行 COBOL 代码仍在生产环境中运行（IBM 估算）
- 约 95% 的美国 ATM 交易运行在基于 COBOL 的系统上
- 传统现代化需要多年、多团队的项目周期
</Aside>

<Aside>
2026 年 2 月，Anthropic 发布了 COBOL 现代化方案，将 Claude Code 定位为遗留咨询团队的直接替代方案。本文描述的工作流适用于任何大型遗留代码库（COBOL、Fortran、VB6、PL/I），而不仅仅是 COBOL。
</Aside>

## 四步工作流

<Aside type="tip" title="独立验证">
学术研究（WJAETS 2025）显示平均可缩减 25-30% 的时间线。最佳案例：Airbnb 在 6 周内迁移了 3500 个测试文件，而预估需要 1.5 年。COBOL 到 Java 的转换准确率在受控研究中达到 93%（arXiv，2025 年 4 月）。
</Aside>

<Steps>

1. **自动化探索与发现**

   映射整个代码库的结构：

   - 识别所有程序入口点和执行路径
   - 追踪跨数百个文件的子程序调用
   - 记录通过共享文件、数据库和全局状态产生的隐式依赖
   - 在触碰任何一行代码之前生成依赖图

   **提示词模式：**
   ```
   "读取整个 [COBOL/遗留] 代码库。映射其结构：
    入口点、执行路径、子程序调用链、
    以及通过共享数据结构、全局变量或
    文件 I/O 产生的任何隐式依赖。输出依赖图。"
   ```

2. **风险分析与机会映射**

   基于依赖图进行分析：

   - 评估模块之间的耦合程度（高耦合 = 高风险）
   - 找出可安全现代化的孤立组件
   - 识别重复逻辑和死代码
   - 将共享状态标记为最高风险区域

   **提示词模式：**
   ```
   "基于依赖图：按耦合程度对模块进行排名。
    哪些组件可以独立现代化？
    哪些与 3 个以上其他模块共享状态，应该最后处理？"
   ```

3. **战略规划**

   人机协作阶段：

   - AI 根据风险/依赖分析建议优先级
   - 团队根据业务优先级进行审查（什么故障最昂贵）
   - 定义目标架构和代码标准
   - 在迁移开始前设计函数级测试进行验证

   <Aside type="caution">
   此阶段**无法完全自动化** -- 业务上下文需要人类判断。混合人机工作流的初始时间估算内完成率比纯自动化方法高 31%（WJAETS 2025）。
   </Aside>

4. **增量实施**

   永远不要一次迁移整个系统：

   - 逐组件翻译逻辑
   - 为仍在使用的遗留组件创建 API 包装器
   - 在生产环境中并行运行新旧代码
   - 在进入下一个组件前独立验证每个组件

   **提示词模式：**
   ```
   "将 [模块 X] 翻译为 [目标语言]。
    保留精确的业务逻辑 -- 暂不优化。
    添加兼容性包装器使两个版本可以并行运行。
    编写测试验证相同输入产生相同输出。"
   ```

</Steps>

## 关键原则

| 原则 | 重要性 |
|------|--------|
| **先映射再动手** | 盲目迁移必然失败；先做发现 |
| **先隔离再迁移** | 高耦合模块 = 级联故障 |
| **并行运行** | 只有两个版本共存时才能回滚 |
| **在边界测试** | 测试输入/输出，而非内部逻辑（内部逻辑会变化） |
| **人工审查业务逻辑** | AI 无法判断哪些边界情况是监管要求还是死代码 |

## 合理预期

"从数年缩短到数季度"是真实的 -- 但这是**乐观场景**，而非平均水平：

| 场景 | 时间线缩减 | 来源 |
|------|-----------|------|
| 保守估计 | -25% 到 -30% | WJAETS 2025 学术综述 |
| 自动化密集阶段 | -40% 到 -50% | Fullstack Labs 行业综合分析 |
| 最佳案例（测试迁移） | -88%（6 周 vs 1.5 年） | Airbnb 案例研究 |
| COBOL 到 Java 转换准确率 | 93% | arXiv，2025 年 4 月 |

平均收益是真实且显著的。标题数字需要有利条件：良好的测试覆盖率、孤立的模块，以及一个同时理解遗留系统和目标技术栈的团队。

## 反模式

<CardGrid>
  <Card title="大爆炸迁移" icon="warning">
    一次重写所有内容。没有公司能在大规模下存活于此。
  </Card>
  <Card title="无并行运行" icon="warning">
    没有回退方案就直接切换。一个未发现的边界情况 = 生产事故。
  </Card>
  <Card title="跳过发现" icon="warning">
    在映射之前就开始翻译。你会破坏你不知道存在的东西。
  </Card>
  <Card title="信任 AI 处理业务逻辑" icon="warning">
    AI 忠实地翻译它读到的内容。如果原始代码是错误的或依赖上下文，翻译也会如此。
  </Card>
</CardGrid>

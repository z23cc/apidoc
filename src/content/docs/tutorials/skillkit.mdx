## 什么是 SkillKit？

SkillKit 是 AI Agent 技能的**开源包管理器**。不同 Agent 使用不同格式——Claude Code 用 `.claude/skills/`，Cursor 用 `.mdc`，Copilot 用 `.github/skills/`。SkillKit 让你**写一次技能，部署到 44 个 Agent**。

<CardGrid>
  <Card title="15,000+ 技能" icon="open-book">
    从精选市场安装技能，涵盖 PDF 处理、代码审查等。
  </Card>
  <Card title="跨平台翻译" icon="puzzle">
    自动在 44 种 Agent 格式之间转换，无需手动改写。
  </Card>
  <Card title="智能推荐" icon="magnifier">
    分析项目技术栈，推荐最相关的技能。
  </Card>
  <Card title="安全扫描" icon="warning">
    46+ 检测规则，安装前扫描提示注入、硬编码密钥等威胁。
  </Card>
</CardGrid>

## 快速开始

<Steps>
1. 初始化项目：
   ```bash
   cd my-project
   skillkit init
   ```
   `init` 会自动检测你的 AI Agent、创建技能目录、保存偏好。

2. 获取推荐：
   ```bash
   skillkit recommend
   ```

3. 安装技能：
   ```bash
   skillkit install anthropics/skills
   ```

4. 部署同步：
   ```bash
   skillkit sync
   ```
</Steps>

### init 创建的内容

| Agent | 技能目录 | 配置文件 |
|-------|---------|---------|
| Claude Code | `.claude/skills/` | `.claude/config.json` |
| Cursor | `.cursor/skills/` | `.cursor/config.mdc` |
| Codex | `.codex/skills/` | `AGENTS.md` |
| GitHub Copilot | `.github/skills/` | — |
| Universal | `skills/` | `AGENTS.md` |

```bash
skillkit init --agent cursor   # 指定 Agent
skillkit init --list           # 显示所有 44 个支持的 Agent
skillkit init --quiet          # 静默模式（CI 用）
```

## 技能生命周期

技能经过 6 个阶段：

| 阶段 | 说明 |
|------|------|
| 发现 | 扫描目录查找 SKILL.md 文件 |
| 质量评估 | 产生 0-100 分（A-F 等级） |
| 解析 | 提取 YAML frontmatter 和 Markdown 内容 |
| 翻译 | 转换为 Agent 特定格式 |
| 安装 | 部署到 Agent 目录并跟踪元数据 |
| 测试 | 14+ 断言类型验证技能 |

### 质量评估

技能在四个维度评分：

| 维度 | 分值 | 评估内容 |
|------|------|----------|
| 结构 | 25 分 | Frontmatter 完整性、Markdown 分节 |
| 清晰度 | 30 分 | 描述质量、指令精确度 |
| 具体性 | 30 分 | 具体示例、可操作步骤 |
| 高级 | 15 分 | 测试覆盖、版本信息、示例 |

等级：A (90-100)、B (80-89)、C (70-79)、D (60-69)、F (低于 60)

```bash
skillkit install anthropics/skills                        # 安装整个仓库
skillkit install anthropics/skills --skills pdf,xlsx      # 指定技能
skillkit install anthropics/skills --agent claude-code,cursor  # 指定 Agent
```

### 安装来源

```bash
skillkit install anthropics/skills          # GitHub
skillkit install gitlab:team/skills         # GitLab
skillkit install ./my-local-skills          # 本地路径
```

### 发布技能

**自托管（推荐）**——生成 RFC 8615 well-known URI 结构：

```bash
skillkit create my-skill          # 创建技能
skillkit validate my-skill        # 验证
skillkit publish ./my-skill --output ./public  # 生成托管结构
```

生成的目录：

```
.well-known/skills/
  index.json              # 技能清单，用于自动发现
  my-skill/
    SKILL.md              # 技能内容
```

部署到你的域名后，用户可以通过 `skillkit add https://your-domain.com` 安装。

**提交到 SkillKit 市场**：

```bash
skillkit publish submit    # 打开 GitHub Issue 等待审核
```

## 跨平台翻译

SkillKit 使用适配器模式在 44 种 Agent 格式间自动转换：

```bash
skillkit translate react-patterns --to cursor     # 单个技能
skillkit translate --all --to windsurf            # 所有技能
skillkit translate my-skill --to codex --dry-run  # 预览模式
```

### 批量翻译

```bash
skillkit agent translate --source ./my-skills --to cursor
skillkit agent translate --source ./skills --to cursor --recursive
```

### 格式映射

| 源格式 | 目标格式 | 说明 |
|--------|---------|------|
| SKILL.md | .mdc | Cursor 格式 |
| SKILL.md | Markdown | GitHub Copilot |
| .mdc | SKILL.md | 反向转换 |

<Aside>
每个适配器实现 `getConfigPath()`（返回技能目录）和 `generateConfig()`（转换内容格式）。
</Aside>

## Primer：自动生成 Agent 指令

Primer 分析代码库，自动为所有 44 个 Agent 生成优化指令：

```bash
skillkit primer                              # 分析当前项目
skillkit primer --agent claude,cursor,windsurf  # 指定 Agent
skillkit primer --all-agents                 # 所有 44 个 Agent
skillkit primer --dry-run                    # 预览不写入
```

### 检测内容

| 类别 | 检测项 |
|------|--------|
| 技术栈 | React、Next.js、Vue、TypeScript、Python 等框架和语言 |
| 构建工具 | Vite、Webpack、Turbopack、esbuild |
| 项目模式 | 架构（Monorepo/微服务）、代码风格、目录结构 |
| 最佳实践 | 测试设置、Linting、文档、Git 工作流 |

### 生成结果

| Agent | 生成文件 |

| Codex | `.codex/instructions.md` |
| + 41 更多 | 各自原生格式 |

## 工作流编排

将技能组合为多步骤流程：

```bash
skillkit workflow list           # 列出工作流
skillkit workflow run <name>     # 运行工作流
skillkit workflow create <name>  # 创建工作流
```

### 工作流定义

```yaml
name: feature-development
description: 完整功能开发工作流
steps:
  - name: plan
    skill: planning
  - name: implement
    skill: coding-standards
  - name: test
    skill: testing-patterns
  - name: review
    skill: code-review
```

### 条件执行和并行执行

```yaml
steps:
  # 条件步骤
  - name: security
    skill: security-audit
    condition: "files.any('**/*.ts')"

  # 并行步骤
  - name: checks
    parallel:
      - skill: linting
      - skill: testing
```

## 测试框架

SkillKit 内置测试框架，支持 14+ 断言类型：

```bash
skillkit test              # 运行所有测试
skillkit test ./my-skill   # 测试指定技能
skillkit test --tags unit  # 按标签过滤
skillkit test --json       # CI 输出格式
```

### 测试文件

在技能目录中创建 `test.yaml`：

```yaml
name: my-skill-tests
skill: my-skill
tests:
  - name: validates input
    input: |
      Process this file
    assertions:
      - contains: "processed"
      - not_contains: "error"
```

### 断言类型

| 断言 | 说明 |
|------|------|
| `contains` | 文本包含 |
| `not_contains` | 文本不包含 |
| `matches` | 正则匹配 |
| `starts_with` | 前缀检查 |
| `ends_with` | 后缀检查 |
| `has_section` | Markdown 分节存在 |
| `has_code_block` | 代码块存在 |
| `word_count_min` / `max` | 词数范围 |
| `no_todos` | 无 TODO 注释 |
| `has_frontmatter` | YAML frontmatter 存在 |

## REST API

启动 HTTP 服务器用于运行时技能发现：

```bash
skillkit serve                    # 启动在 http://localhost:3737
skillkit serve --port 8080        # 自定义端口
```

### 主要端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/search?q=...` | 搜索技能 |
| POST | `/search` | 带过滤器搜索 |
| GET | `/skills/:owner/:repo/:id` | 获取特定技能 |
| GET | `/trending?limit=20` | 热门技能 |
| GET | `/categories` | 技能分类 |
| GET | `/health` | 服务器健康检查 |
| GET | `/docs` | Swagger UI |
| GET | `/openapi.json` | OpenAPI 规范 |

### 排名算法

| 信号 | 分值 | 说明 |
|------|------|------|
| 内容 | 40 分 | 有描述和 SKILL.md 内容 |
| 查询匹配 | 30 分 | 名称/描述与搜索词的匹配度 |
| 流行度 | 15 分 | GitHub stars + 安装量（对数缩放） |
| 引用 | 15 分 | 有示例文件、文档或资源 |

限流：每 IP 每分钟 60 次请求。内置 LRU 内存缓存。

## 配置

SkillKit 使用 5 层配置（优先级从低到高）：

1. 代码中的默认值
2. 全局用户偏好 (`~/.skillkit/preferences.json`)
3. 项目设置 (`skillkit.yaml`)
4. 环境变量
5. CLI 标志

### 项目配置

在项目根目录创建 `skillkit.yaml`：

```yaml
version: 1
agent: cursor              # 覆盖自动检测
autoSync: true             # 变更时自动同步
enabledSkills:
  - pdf
  - xlsx
  - react-patterns
sources:
  - anthropics/skills
  - vercel-labs/agent-skills
```

### 环境变量

| 变量 | 用途 |
|------|------|
| `SKILLKIT_AGENT` | 覆盖默认 Agent |
| `SKILLKIT_DIR` | 自定义 SkillKit 目录 |
| `SKILLKIT_DEBUG` | 启用调试日志 |

### 配置文件

```bash
skillkit settings --set agent=cursor
skillkit settings --set autoSync=true
skillkit settings --get agent
```

### 配置 Profile

为不同环境创建不同配置：

```yaml
profiles:
  development:
    agent: claude
    autoSync: true
  production:
    agent: universal
    autoSync: false
```

```bash
skillkit --profile production sync
```

## Chrome 扩展

从浏览器直接将网页保存为技能：

<Steps>
1. 构建扩展：`pnpm --filter @skillkit/extension build`
2. Chrome → `chrome://extensions` → 加载已解压的扩展 → 选择 `packages/extension/dist/`
3. 点击扩展图标或右键 → "Save page as Skill"
</Steps>

扩展将页面 URL 发送到 SkillKit API 进行服务端提取，生成的 SKILL.md 自动下载。然后运行 `skillkit install ~/Downloads/skillkit-skills/my-skill` 部署到所有 Agent。
